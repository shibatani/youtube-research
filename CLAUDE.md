# youtube-research

福岡特化のゆっくり解説 YouTube チャンネル「福岡井戸端会議」(@fukuoka-idobatakaigi) 運営のためのリサーチツール集。

## チャンネルの本質

**福岡県民の自分事を、ネガティブ・優越感・恐怖などの強い感情でドライブして届ける、エンタメ×地元郷土型チャンネル。**

都道府県特化チャンネルの強みは「狭く深く刺す」設計。同じ深さでも感情強度の高い切り口を選ぶことで優先度を上げさせる。

## 企画評価の3軸

新企画・ライバル動画のスクリーニングは以下の3軸の**掛け算**で評価する。どれか1つが0だと死ぬ。スコアは各 1〜3点、合計 3〜9点。

### 1. 自分事化の濃度 (1〜3点)

視聴者が「自分ごと」と感じる度合い(認知フィルタ通過)。

- **3点**: 全県民が反応する — 自分の街・母校・住所(地名)・治安・転職先候補
- **2点**: 条件付きで反応する — 自分の会社/業界・思い出ある場所
- **1点**: 対象事化/他人事 — 自分じゃない・倒産済み・関係ない場所

### 2. 射程の広さ (1〜3点)

ターゲットのパイ。

- **3点**: 広い — 全年齢×全県民が当てはまる(街・地名・治安)
- **2点**: 中 — 特定の世代/属性のみ(社会人・進学経験者)
- **1点**: 狭い — ニッチ層のみ(50代以上の懐古層など)

### 3. 覚醒度 / Arousal (1〜3点)

Russell円環モデル(1980)の **Arousal(行動準備性)** 軸。視聴者に「今すぐクリック指が動く」と思わせる感情ドライブの強さ。

**重要**: Valence(快-不快の強さ)とは独立軸。「深い悲しみ」は Valence が強いが Arousal が低いのでクリックされない。**判定は Arousal 一本で行う**。

- **3点**: 高覚醒 — 恐怖・不安・優越感・嫉妬・怒り・歓喜・驚き(「ヤバいかも」と思う)
- **2点**: 中覚醒 — 「関係ある」と思う
- **1点**: 低覚醒 — 安らぎ・感心・教養・しんみり・深い悲しみ(「へぇ」で終わる)

**企画チェック質問**: 「ネタを見た瞬間に視聴者は『自分も関係あるかも』『これヤバいかも』と思うか?」Yes なら高覚醒。

## 4象限マトリクス(自分事化×射程のスクリーニング)

| 象限 | 戦略的意味 | 結果レンジ | 代表例 |
|---|---|---|---|
| **広×深** | ホームラン狙いゾーン(主軸) | 5万以上の大ヒット | 住みたくない街・母校偏差値 |
| **広×浅** | 中ヒット量産ゾーン(補助) | 1〜4万 | 怖い地名・勝ち組企業・日本一・方言 |
| **狭×深** | ファン化ゾーン(月1本のスパイス) | 数千〜2万 | 消えたチェーン店 |
| **狭×浅** | 死ぬ(原則やらない) | 数百 | 屋台の歴史 |

**優先順位(配分比率)**:
1. 広×深 → 6〜7割(主軸)
2. 広×浅 → 2〜3割(補助・量産ベース)
3. 狭×深 → 月1本(ファン化用)
4. 狭×浅 → 原則やらない

**戦略の核**: 「広×深×ネガティブ感情」を軸に企画し、外しても広×浅で着地できる実装をする。

## 企画フェーズの2段階構造

**企画 = 動画の切り口とワードを決めるところまで**。以下の順序を守る。

1. **Step 1: 3軸評価(スクリーニング)** — 自分事化 × 射程 × 覚醒度で切り口を篩にかける
2. **Step 2: ワード確定** — 勝ち残った切り口を、最も覚醒度が高い言い回しで確定する
   - 例: 「住みやすい街」(低覚醒) < 「住みたくない街」(不安) < 「地獄の街」(恐怖)

実装フェーズ(サムネ・タイトル・台本・編集・BGM・タイミング)は企画の後工程。

## 重要な原則

- **サムネ詐欺はしない** — 維持率を守る、次の動画を殺す唯一の確実な要因
- **素材評価と実装評価を分離する** — 3軸は企画素材の評価であって最終結果の予測ではない
- **後付け分析を避ける** — 12個ぐらいの変数が掛け算で結果を決めるため線形分解は不可能。ヒット動画の共通項を抽出して再現することにフォーカス
- **「自分が他県民だったらこのタイトルでクリックするか?」** で自己検証する

## ライバル分析 — 手動フロー(2段階)

- **対象**: 他県特化系ゆっくり解説チャンネル(沖縄・北海道・九州・千葉・愛知・埼玉)
- **出力**: スプシ `1cJ5Enyy-yiyx3YKM1uwVUZz5LKqsnRvaaaAHUayIs-8` のシート「ライバル分析」
- **並び順**: 合計スコア降順 → 再生回数降順
- **必要**: `YOUTUBE_API_KEY` + `credentials.json` (Sheets サービスアカウント)

### フロー
1. `yarn rival:fetch` — YouTubeから全ライバル動画+自チャンネル過去タイトル+シート既存評価を取得し、`rival-pending.json` に書き出す
2. **Claude Code セッション上で新規動画を3軸評価** — サブエージェント(Explore など)に本CLAUDE.mdの3軸定義と自チャンネル過去タイトルを渡して評価を委譲、結果を `newEvaluations[videoId] = { alreadyCovered, jibungotoka, shatei, kakuseido }` の形で JSON に埋める
3. `yarn rival:apply` — 既存評価(キャッシュ)と新規評価をマージし、`alreadyCovered=true` は除外、合計スコア降順でスプシに書き戻す

### 評価のポイント
- 他県の動画でも **「福岡版で同じ切り口を扱うとしたら何点?」** の視点で評価する
- `alreadyCovered` は自チャンネルの過去タイトルと切り口が同じなら true(ワード違いでも同企画なら true)
- 評価は1〜3点(各軸)、合計 3〜9点
